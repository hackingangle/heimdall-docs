# Heimdall 素材收集任务技能（heimdall-collect-skill）

> 供 Claude Code 等外部 Harness 作为技能装载。把一期节目的选题变成一个可执行、可验收、可续跑的**收集任务**：调研简报 → 分轮执行 → 缺口盘点。

- 日期：2026-07-06
- 前置技能：[heimdall-material-skill](./heimdall-material-skill.md)（素材 API 与入库质量规则）
- 环境自检：[heimdall-doctor-skill](./heimdall-doctor-skill.md)（首次执行或平台连通问题时先跑）
- 相关设计：[素材收集任务设计](../superpowers/specs/2026-07-06-collect-task-design.md)

> **获取手段不做任何假定**：搜索、抓取、字幕、PDF、用户提供的文件……由执行模型按当下可用能力自行选择与组合。本技能只约束**任务怎么组织**（计划、分轮、盘点），入库质量由 material 技能约束。

## 1. 任务定义

输入：一个选题（如「某历史人物传记」）+ 目标项目 ID。
输出：

1. 一条 `[调研计划]` 素材（任务合同）；
2. 分轮收集入库的一批素材（获取手段由执行模型自选）；
3. 一条持续更新的 `[调研索引]` 素材（逐维度缺口盘点，任务进度即在此）。

**终止条件明确**：调研计划里的各维度全部达标，任务完成。不是"感觉差不多了"。

## 2. 第一步：调研简报（开搜之前必做）

直接拿主题去搜是大忌——查询词同质化，覆盖面窄。先把主题拆成 **4~8 个挖掘维度**，每个维度写清四件事：

| 要素 | 说明 |
|---|---|
| 想挖什么 | 这个维度要回答的问题、要拿到的故事/观点 |
| 查询词组 | 3~5 组不同角度的搜索词（含 site: 限定、书名、人名等） |
| 预期载体 | text / pdf |
| 达标条件 | 可判断的标准，如「年谱全本入库（text 或 pdf）」「至少 2 个弟子回忆视频字幕（text）」 |

### 人物类选题的默认维度骨架（可增删）

1. **生平骨架**：年谱、大事记、时间线；
2. **第一手声音**：自述、开示/演讲、书信、著作；
3. **传奇故事点**：具体事件，一事一挖；
4. **关系人回忆**：弟子、亲友、同时代人的讲述；
5. **时代背景**：人物所处的历史/社会语境；
6. **观点与争议**：后人评价、不同立场的看法（做观点对撞用）。

其他题材（概念科普、事件复盘等）由执行 Agent 自拟维度，逻辑相同。

### 计划入库与确认

- 简报写成一条素材入库：`type=text`，标题 `[调研计划] {主题} - {YYYY-MM-DD}`；
- **入库后向用户过目维度拆解再开跑**（维度错了后面全白搭）；用户明确说"直接跑"则跳过确认；
- 计划实例范本见 [examples/research-plan-example.md](./examples/research-plan-example.md)。

## 3. 第二步：分轮执行

**每轮只跑 1~2 个维度**，不要一把梭（单会话上下文有限，全跑完后半程清洗质量会掉）。

- 每轮针对当轮维度自行选择获取手段与来源组合；核心著作类来源（如人物的年谱、文集）往往是定海神针，优先处理。
- 执行纪律：相关性速读判断，无关直接丢弃（记入调研索引）；单条失败不中断，记录原因继续；反爬/付费墙来源不硬啃，记入索引留人工。
- 入库规则（标题格式、出处头、`origin_url`/`raw_content` 留档、查重）全部沿用 material 技能，此处不重复。
- 轮与轮之间允许用户翻看已入库素材纠偏；用户提出的调整写回调研计划素材（PATCH）。

## 4. 第三步：缺口盘点（每轮收尾必做）

维护一条 `[调研索引] {主题}` 素材（`type=text`）。**同一任务只有一条索引，每轮 PATCH 更新，不要新建**。内容三部分：

1. **维度状态表**（对照调研计划逐维度打勾）：

```markdown
| 维度 | 状态 | 已收素材 | 缺口与下一轮建议 |
|---|---|---|---|
| 生平骨架 | ✅ 达标 | 年谱全本 ×1 | — |
| 关系人回忆 | 🟡 部分覆盖 | 弟子访谈字幕 ×1 | 还差其他弟子方向，建议换关键词补充 |
| 观点与争议 | ⬜ 未开始 | — | 第三轮处理 |
```

2. **失败/放弃清单**：抓不到的 URL、无字幕的视频、判定无关而丢弃的内容，附原因；
3. **人工补充建议**：Agent 手段覆盖不了的方向（微信公众号、纸书扫描等）。

## 5. 续跑协议

任何新会话/新 Agent 接手任务时：

1. `GET /projects/{id}/materials` 找到 `[调研计划]` 与 `[调研索引]` 两条素材并读取；
2. 从索引的维度状态表挑未达标维度继续执行；
3. 不依赖任何会话历史。

## 6. 收尾

全部维度达标后，最后一次更新调研索引（所有维度标 ✅ 或注明放弃原因），并向用户汇报：各维度收录统计、素材类型分布、遗留人工事项。
