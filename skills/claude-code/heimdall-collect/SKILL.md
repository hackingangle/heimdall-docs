# Heimdall 收集任务技能

选题 → 素材库的完整编排。本技能管**任务怎么组织**；入库 API 与质量规则见 heimdall-material 技能。

**获取手段不做任何假定**：搜索、抓取、视频字幕、PDF、用户提供的文件……按当下可用能力自行选择与组合，用你认为最有效的手段拿到素材。

## 0. 判断任务状态（每次激活先做）

查项目素材列表，看是否已存在本主题的 `[调研计划]` 素材：

- **不存在** → 新任务，从第 1 步开始；
- **已存在** → 续跑：读取 `[调研计划]` 和最新的 `[调研索引]`，从索引中未达标的维度继续（跳到第 2 步）。续跑不依赖任何会话历史。

## 1. 调研简报（开搜之前必做）

**禁止直接拿主题去搜**——查询词同质化，覆盖面窄。先拆维度：

- 把主题拆成 **4~8 个挖掘维度**，每个维度写四件事：想挖什么 / 3~5 组不同角度的查询词 / 预期载体（text、pdf）/ **可判断的达标条件**（如「年谱全本入库」「至少 2 位弟子回忆」）。
- 人物类选题默认骨架（可增删）：生平骨架、第一手声音（自述/开示/书信）、传奇故事点（一事一挖，每事 2 个来源互证）、关系人回忆、时代背景、观点与争议。其他题材自拟。
- 附**分轮安排**：每轮 1~2 个维度，注明重点来源方向。
- 简报作为素材入库：`type=text`，标题 `[调研计划] {主题} - {YYYY-MM-DD}`。
- **入库后停下来，向用户过目维度拆解再开跑**；用户明确说"直接跑"则跳过确认。

## 2. 分轮执行

**每轮只跑当轮维度，不要一把梭**（一次跑完后半程清洗质量会掉）。每个维度：发现 → 获取 → 速读判断相关性 → 清洗 → 入库（走 heimdall-material 规则）。

### 执行纪律

- 获取手段自选，但产出有质量底线：素材保留原文不过度摘要，一本书/一篇演讲/一篇文章/一个字幕 = 一条素材（书不按章拆条，章节用正文 Markdown 标题保留）；
- 核心著作类来源（如人物的年谱、文集）往往是定海神针，优先处理；
- 相关性速读判断，无关直接丢弃（记入调研索引）；
- 单条失败不中断，记录原因继续；反爬、付费墙来源不硬啃，记入索引留人工；
- 轮与轮之间用户可能纠偏，调整写回 `[调研计划]`（PATCH）。

## 3. 缺口盘点（每轮收尾必做）

维护一条 `[调研索引] {主题}` 素材（`type=text`）。**同一任务只有一条索引，每轮 PATCH 更新，不要新建**。内容：

1. **维度状态表**（对照计划逐维度）：

```markdown
| 维度 | 状态 | 已收素材 | 缺口与下一轮建议 |
|---|---|---|---|
| 生平骨架 | ✅ 达标 | 年谱全本 ×1 | — |
| 关系人回忆 | 🟡 部分覆盖 | 弟子访谈 ×1 | 还差其他关系人方向 |
| 观点与争议 | ⬜ 未开始 | — | 第三轮处理 |
```

2. **失败/放弃清单**：URL + 原因；
3. **人工补充建议**。

## 4. 终止条件与收尾

各维度全部 ✅（或注明放弃原因）即任务完成——不是"感觉差不多了"。最后一次更新索引后向用户汇报：各维度收录统计、类型分布、遗留人工事项。
