# Heimdall 素材收集任务设计（heimdall-collect 技能）

- 日期：2026-07-06（同日更新：吸收"不假定获取手段"决策）
- 状态：已确认
- 前置：heimdall-material 技能（素材 API 与入库质量规则）
- 原则一：机制与策略分离——收集任务是「策略之上的编排」，仍活在 Agent 侧，后端零改动
- 原则二：**不假定 Harness 如何获取素材**——搜索、抓取、字幕、PDF、用户提供的文件，信任模型按当下可用能力自选手段。因此不设 research（获取手段）技能，[原三通道设计](./2026-07-05-material-research-design.md)已废弃；技能体系只约束两层：任务组织（collect）+ 入库质量（material）。

## 1. 背景与问题

收集实践暴露出**任务层缺失**（无论用什么手段获取）：

1. **查询词同质化**：直接拿主题去搜，多次搜索容易撞车，覆盖面窄；
2. **一把梭质量衰减**：单次会话把所有方向跑完，上下文膨胀，后半程清洗质量明显下降；
3. **没有终止条件**："感觉差不多了"就停，缺什么、够不够写稿，无人回答；
4. **不可续跑**：换会话/换 Agent 接手时，不知道前面收了什么、还差什么。

## 2. 方案决策

把「收集任务」定义为三段式编排，落为一个新技能 `heimdall-collect`，编排层只依赖素材 API，获取手段由执行模型自选：

1. **调研简报（计划）**：开搜之前，先把主题拆成 4~8 个**挖掘维度**，每个维度写明想挖什么、查询词、预期载体与达标条件。简报本身作为一条 `type=text` 素材入库（标题 `[调研计划] {主题} - {日期}`），是任务的"合同"。
2. **分轮执行**：每轮只跑 1~2 个维度，获取手段与来源组合由模型按维度自定；轮与轮之间允许人工翻看素材纠偏。入库规则沿用 material 技能。
3. **缺口盘点（升级版调研索引）**：每轮收尾产出/更新 `[调研索引]` 素材，除"收了什么、什么失败"外，必须**对照调研计划逐维度打勾**：已达标 / 部分覆盖 / 空缺，并给出下一轮建议。各维度达标即任务完成——这是明确的终止条件。

## 3. 关键规则

- **计划先行，用户确认**：维度拆解产出后向用户过目再开跑（维度错了，后面全白搭）；用户明确说"直接跑"则跳过确认。
- **续跑靠读库**：任何新会话接手任务，先读 `[调研计划]` 与最新 `[调研索引]` 两条素材即可恢复全部上下文，不依赖会话历史。
- **调研索引每轮更新**：同一任务只维护一条索引素材（PATCH 更新），不要每轮新建一条。
- **人物类选题的维度骨架**（可复用的默认拆法）：生平骨架 / 第一手声音（自述、开示、书信）/ 传奇故事点 / 关系人回忆（弟子、亲友）/ 时代背景 / 观点与争议。其他题材由 Agent 自拟。

## 4. 落地形式

- Claude Code 版：`docs/skills/claude-code/heimdall-collect/SKILL.md`（安装到 `~/.claude/skills/heimdall-collect/`）

## 5. 验收

用一期真实选题跑完整流程：计划入库 → 至少两轮执行 → 索引中各维度均有明确状态；抽查「续跑」场景（新会话仅凭两条素材恢复任务）。

## 6. 本期不做

服务端任务实体（task 表 + 状态机）、定时/自动触发收集、多 Agent 并行编排。验证两三期后再评估产品化。
